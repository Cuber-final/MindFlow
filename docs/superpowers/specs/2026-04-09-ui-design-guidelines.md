# MindFlow UI Design Guidelines

> 状态：当前前端统一设计规范
> 适用范围：`frontend/src/` 全部页面与组件
> 目的：将产品需求、现有实现风格与后续设计方向收敛为一套可执行的 UI 原则

---

## 1. 产品定位

MindFlow 不是资讯门户，也不是传统后台系统。

它是一个面向个人私有化部署的 AI 简报工作台：
- 抓取用户选择的信息源
- 提炼有判断力的锚点内容
- 结合兴趣权重生成每日简报
- 通过显式与隐式反馈持续学习用户偏好

因此，UI 的核心任务不是“展示尽可能多的信息”，而是：
- 让用户高效阅读每日简报
- 让用户理解推荐逻辑和兴趣演化
- 让配置行为保持低摩擦、可信、可控

一句话定义：

**MindFlow 应该像一份由私人编辑为你整理的每日世界观察，而不是一个拥挤的信息管理后台。**

---

## 2. 设计总原则

### 2.1 North Star

统一设计方向为：

**Editorial Workbench**

关键词：
- 编辑感
- 纸感
- 冷静
- 可信
- 有判断力
- 轻工具感

这意味着界面必须同时具备两种气质：
- 阅读页像 Newsletter / 编辑精选
- 管理页像同一出版系统下的配置工作台

不是两套风格拼接，而是一套视觉语言在不同任务密度下的延展。

### 2.2 体验优先级

按优先级排序：

1. 阅读沉浸感
2. 信息层次清晰
3. 操作可信且克制
4. 个性化逻辑可感知
5. 管理能力完整

### 2.3 必须遵守的界面判断标准

如果一个设计方案更像：
- SaaS 仪表盘
- 默认管理后台模板
- 资讯流瀑布页
- 花哨的 AI 产品落地页

则不符合 MindFlow 的产品气质。

如果一个设计方案更像：
- 私人编辑整理的晨间简报
- 研究者的阅读工作台
- 一份有秩序的纸质刊物被数字化重构

则方向正确。

---

## 3. 视觉语言

### 3.1 基础气质

整体视觉应保持：
- 暖白背景，而不是纯白高亮背景
- 高对比正文，但不过度黑硬
- 少量高价值强调色，而不是多主色竞争
- 大量依靠留白、字重、分隔和对齐建立层次
- 尽量减少无意义卡片堆叠

### 3.2 色彩原则

当前方向是正确的，应继续沿用并统一：
- 背景：暖白、纸感米白
- 正文：深灰黑
- 强调：编辑橙红
- 分区辅助：探索区用偏蓝或偏灰蓝
- 状态色：仅在反馈、成功、错误时出现

颜色使用规则：
- `accent` 只用于真正重要的强调：标题 hover、选中态、关键按钮、锚点提示
- 区域色只用于内容分层，不用于全局主按钮泛滥
- 删除、失败、警告等系统状态不要污染主阅读区

### 3.3 字体原则

统一采用：
- 标题：衬线体，承担“编辑感”
- 正文：无衬线体，承担可读性
- 数值/标签/技术信息：等宽或半等宽风格

字体使用规则：
- 页面标题、简报标题、洞察标题使用 display font
- 管理页标题可以降低戏剧性，但不能退化为完全普通的后台标题
- 表单正文、说明、元信息保持简洁克制

### 3.4 阴影与边框

阴影应轻，不做浮夸悬浮感：
- 默认使用极轻阴影或纯边框
- hover 时增加一点层次即可
- 阅读页优先用边线、分隔、缩进建立结构，不要大量卡片

边框规则：
- 阅读内容优先竖线、分节线、弱边框
- 管理模块可使用轻卡片，但边框应柔和、圆角克制

---

## 4. 布局原则

### 4.1 全局布局

全站采用“阅读主列 + 轻侧栏 / 工具区”的结构思维。

推荐布局逻辑：
- 简报页：主内容优先，侧栏服务筛选与上下文
- 管理页：内容列更集中，不做过宽的大后台网格
- 设置页：保持表单宽度收敛，避免横向过散

### 4.2 内容宽度

阅读内容必须控制在舒适宽度内：
- 洞察正文不要铺满超宽屏
- 导语和辩证分析保持阅读节奏
- 管理页表单和列表也应控制最大宽度，体现“工作台”而非“系统台账”

### 4.3 留白策略

留白是层次工具，不是装饰：
- 标题与正文之间要有明确呼吸感
- 分区之间留白应显著大于区块内部间距
- 管理页也应有分组留白，避免表单和卡片挤成一团

---

## 5. 页面风格指南

### 5.1 简报页 `Newsletter`

这是品牌气质的核心页面，必须最像“私人编辑精选”。

页面目标：
- 快速进入当天主题
- 清晰浏览领域分组
- 在一屏内理解文章量、锚点量、日期和兴趣过滤关系

视觉要求：
- 页面头部以标题、日期、导语为核心
- 内容主体弱卡片化，强调分组和阅读节奏
- 洞察块使用左侧边线、标题层级、标签和来源信息组织内容
- 辩证分析应像页边批注或编辑按语，而不是普通提示框

禁止：
- 把简报主体做成密集卡片瀑布流
- 在阅读主区塞过多彩色操作按钮
- 让侧栏视觉权重高于简报内容

### 5.2 兴趣页 `InterestSettings`

这是“用户如何塑造下一份简报”的解释界面。

页面目标：
- 让用户理解兴趣标签与推荐权重关系
- 能快速增删、冻结、激活标签
- 能通过统计感知学习系统在工作

视觉要求：
- 保持管理属性，但沿用 editorial 色彩与字体
- 统计卡需要克制，像摘要信息，不像 KPI 仪表板
- 权重条和分区标签是核心视觉元素
- 推荐标签区应更像“编辑建议补充主题”，而不是系统推荐位

禁止：
- 使用过度商业化的数据看板视觉
- 把每个标签做成过重的运营卡片

### 5.3 新闻源页 `Sources`

这是“内容供应链配置页”，应是工具感最强的页面，但仍属于同一产品语言。

页面目标：
- 快速管理新闻源
- 低成本完成添加、编辑、抓取、删除
- 清楚感知来源类型和抓取状态

视觉要求：
- 保持模块化卡片或列表是合理的
- 但按钮、标题、边框和留白仍要延续主设计语言
- 重点是清晰和可信，不是营销式的彩色按钮堆砌

建议风格：
- 用更克制的按钮层级
- 用状态文本和时间信息增强可靠感
- 强调“来源库”而非“资源管理面板”

### 5.4 设置页 `Settings`

这是“系统可信配置页”，必须简洁、稳重、低噪音。

页面目标：
- 配置 AI 提供商与模型
- 测试连通性
- 查看系统说明和入口链接

视觉要求：
- 表单区域应像编辑后台中的配置章节
- 成功/失败反馈要明确，但不要过度鲜艳
- 分组标题要清楚，区块不要太碎

禁止：
- 通用后台表单模板感过强
- 主按钮、次按钮、辅助链接都抢视觉注意力

## 6. 组件规范

### 6.1 导航

导航应是“刊物目录”式，而不是“管理系统标签页”式。

要求：
- 文字优先于图标
- 图标只做辅助，不做品牌记忆点主体
- 当前选中态更像被编辑标注，而不是高亮导航按钮

### 6.2 洞察块

洞察块是整个产品最重要的内容单元。

必须具备：
- 明确标题层级
- 稳定的正文节奏
- 轻量但可识别的分区徽标
- 来源与标签作为底部元信息
- hover 时才逐步显露更多交互线索

### 6.3 辩证分析框

辩证分析不是普通摘要，不应只是灰色信息框。

应当表现为：
- 编辑批注
- 论点拆解
- 观点的支持 / 质疑 / 延伸结构

推荐表现：
- 分段而非一整坨文本
- 颜色分层明确但克制
- 与正文形成次一级阅读节奏

### 6.4 标签与分区

标签系统承担解释推荐逻辑的职责。

规则：
- 标签应轻，不要像促销 badge
- 分区标签应稳定映射主航道 / 探索区 / 惊喜箱
- 相同语义在全站保持一致颜色和文案

### 6.5 按钮

按钮层级建议统一为三层：
- 主按钮：强调色，数量严格受控
- 次按钮：浅底或描边
- 文本按钮：用于次级跳转与筛选

不要在同一视区出现多个同权重主按钮。

---

## 7. 动效原则

MindFlow 的动效应服务于阅读和理解，不服务于炫技。

推荐动效：
- 页面进入：轻微 fade + upward reveal
- 卡片或洞察 hover：边框/阴影/标题色轻微变化
- 筛选和分区切换：柔和过渡
- 下拉或弹层：短距离展开，不做强烈弹跳

禁止：
- 大幅缩放
- 高频闪烁
- 无意义持续动画

动效语气应为：
- 安静
- 稳定
- 精准

---

## 8. 文案与语气

文案风格应统一为：
- 冷静
- 有编辑判断
- 不夸张
- 不产品经理腔

推荐：
- “今日简报”
- “辩证分析”
- “探索区”
- “推荐标签”
- “上次抓取”

避免：
- “智能赋能”
- “一键提升效率”
- “爆款内容”
- 过强的营销式提示语

---

## 9. 当前实现的风格判断

### 9.1 已经对齐的部分

当前前端已经具备以下正确方向：
- 暖白纸感背景与深色正文
- 衬线标题 + 无衬线正文的基础层次
- 简报页的分组、左侧边线、标签筛选
- 兴趣页中的权重条、区域标签、统计摘要

这些都应保留并继续深化，而不是推倒重来。

### 9.2 当前不够统一的部分

当前不统一主要体现在：
- 顶部导航更像常规后台 tab
- `Sources` 页按钮颜色较散，偏通用后台
- `Settings` 页是标准表单卡片风，编辑感较弱
- 不同页面对同一种品牌气质的表达强度不一致

后续优化方向不是“做得更花”，而是：
- 让所有页面都更像同一份刊物体系下的工具界面
- 让阅读页与管理页共享同一种设计语法

---

## 10. 后续前端改版执行规则

后续如继续改 UI，按以下顺序执行：

1. 先保证简报页成为全站视觉母版
2. 再让兴趣页继承相同的字体、留白和状态语言
3. 再收敛新闻源页与设置页的按钮、边框、标题层级
4. 最后统一导航与全局壳层

评审标准只有一个：

**新页面接入后，用户是否会觉得它仍然属于同一本“私人编辑简报系统”。**

如果答案不明确，就说明设计还不够统一。
